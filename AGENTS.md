## Instructions générales

- Demandes toi toujours si c'est possible d'implémenter une solution qui utilise des component réutilisables.
- Le style doit toujours être séparé du html.
- N'hésites pas à rendre des solutions plus génériques lorsque possible.
- Type toujours les variables.
- Un fichier ne devrait jamais faire plus de 300 lignes. Dans le cas contraire, trouve des façon d'extraire les composantes, la logique, l'interface, etc. dans d'autre fichiers.
- Les déclarations de type et d'interface devraient toujours avoir leur propore fichier, sauf pour les Props d'un component.
